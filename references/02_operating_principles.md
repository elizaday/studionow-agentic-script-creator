# Operating Principles

## Principle 1: Diagnose before drafting
Do not jump from prompt to script.
Find the actual job first.

## Principle 2: Solve the assignment that exists
Do not rewrite the brief into the version you wish you got.

## Principle 3: Every script needs an engine
A script should not be “a montage of relevant things.”
It should have an organizing mechanism:
- relay
- route
- reveal
- countdown
- before/after contrast
- journey
- day-in-the-life
- system walkthrough
- chaptered case study
- sensory escalation
- thesis across time
- modular city swap
- proof ladder

## Principle 4: Generic language is not a substitute for thinking
If the line could fit fifty other brand videos, it is probably weak.

## Principle 5: Production reality matters
Write with awareness of:
- runtime
- asset access
- talent and likeness
- archival rights
- location access
- approvals
- AI limitations
- what is locked vs variable

## Principle 6: Fewer stronger choices beat more vague choices
Specificity is usually better than coverage.
If the brief asks for too much in too little time, solve structurally.

## Principle 7: Match tone to the real use case
Not everything is an anthem.
Not everything is a sizzle.
Not everything needs VO.
Not everything needs a big emotional ending.

## Principle 8: Build for review and production
The script has to persuade stakeholders and guide makers.
It cannot only do one of those jobs.
